# Задание 1
## Стратегия масштабирования и отказоустойчивости.
1.	Географически распределенная инфраструктура. В каждой географической зоне отдельный Kubernetes кластер и кластер БД PostgreSQL. Зоны распределяем по часовым поясам. 
2.	Для синхронизации данных используем кластер PostgreSQL. Внутри зоны доступности реплики кластера синхронные. Между географическими зонами реплики асинхронные.
3.	Авто масштабирование подов будем осуществлять на основании метрик нагрузки.
## Конфигурацию развёртывания приложения в Kubernetes.
В каждой географической зоне независимый кластер Kubernetes. Использование одного растянутого кластера Kubernetes требовательно к каналам связи и может привести к деградации всей системы, в случае долгосрочного обрыва связи между зонами доступности. 
## Балансировка нагрузки.
Для балансировки нагрузки используем GSLB. Трафик между географическими зонами распределяем на основе IP геолокации пользователя.
## фейловер-стратегия
Для обеспечения требования к доступности 99,9% используем стратегию Active-Active.
## Конфигурация базы данных
Репликация Master-Master
Необходимо обеспечить локальные операции записи и чтения в каждом регионе для снижения задержек.
Повышение отказоустойчивости: при выходе из строя одного узла другие продолжают полнофункциональную работу.
Возможность масштабирования операций записи.

Для оптимизации доступа к данным будем использовать отдельно реплики для чтения и реплики для записи.
В каждой зоне доступности настроен отдельный контур мониторинга и логироваия. А также асинхронный сбор логов в централизованное хранилище, что позволит оценить работу всей системы в целом.
